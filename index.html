Core

The Metadata Secret

The Metadata is a repository that stores MicroStrategy object definitions and information about your data warehouse. It allows for object reusability and impact analysis.

  • Consistency: Define once, use everywhere.
  • Security: Centralized ACL management.
  • Portability: Move projects between environments easily.
Role: Architect

Architecting the Vision

The Architect's primary mission is to translate physical warehouse structures into a logical business model that users can understand.

Logical Data Model

Abstracting the DB schema into business entities.

Relationship Mapping

Defining 1:1, 1:M, and M:M relationships between tables.

Architect

Schema Objects: Facts

Facts are the foundation of metrics. They represent numeric data stored in the warehouse that can be aggregated.

Fact Expressions

Can be simple column mappings or complex expressions: (Qty * Unit_Price) - Discount

Architect

Schema Objects: Attributes

Attributes provide the business context for facts (the "Who, What, Where, When").

Attribute Forms

ID, Description, Long Description, Image, etc.

Elements

The actual data values (e.g., "New York" is an element of "City").

Architect

Hierarchies & Drills

Hierarchies define the paths users take when drilling down into data (e.g., Year -> Month -> Day).

System Hierarchy

Automatically created based on attribute relationships.

User Hierarchy

Custom-defined paths for specific business needs.

Metrics: Power of Calculation

Simple Metrics

Basic aggregations like Sum(Revenue).

Level Metrics

Calculated at a specific dimensionality (e.g., Revenue for the whole category regardless of current filter).

Compound Metrics

Formulas involving other metrics: (Profit / Revenue).

Developer

Filters: Precision Analytics

Filters restrict the data returned from the warehouse to specific criteria.

Attribute Qualification

Region = Southeast

Metric Qualification

Revenue > 1M

Filter Panels

Dynamic selectors in Dossiers.

Developer

Prompts: User Interactivity

Prompts ask the user for input before the report runs, making the BI experience dynamic and personalized.

  • Object Prompts: User picks the metrics or attributes.
  • Value Prompts: User enters a specific date or number.
  • Element List Prompts: User picks from a list (e.g., Year).
